# 列表功能定义

## 列表与内容类型

- 列表在新增的时候需要指定内容类型
- 列表的字段是从内容类型中来的（继承）
- 列表的默认视图是显示所有的字段
- 列表的扩展字段、视图、表单布局配置统一保存在 `List` 模型的 `schema` JSON 字段中，不再使用独立的 `list_fields` 和 `list_views` 表


## 列表页面布局

- 进入列表页面后，全局设置菜单中出现列表设置菜单项，点击后调整到列表设置页面
- 进入列表后，显示默认视图，在url中使用视图ID进行区分不同的视图
- 列表的上方是操作栏，分为左右2个区域、左侧是新增按钮，批量删除，最右侧是切换视图下拉列表，下拉列表的最后一个选项是编辑当前视图
- 操作栏的下方是列表的数据呈现，使用`vxe-table` 呈现
- 列表表格的最左侧是选择列，固定在最左侧（fixed="left"）
- 列表表格的列根据当前视图配置的字段顺序显示，列上有排序（如果指定了某个字段的排序则临时覆盖视图的排序, 刷新页面后恢复视图排序），筛选功能（TODO:暂时统一按字符串处理）
- 列表的最右侧是操作列，包含查看、编辑、删除按钮，点击查看后跳转到列表数据项的查看页面，点击编辑进入列表数据项的编辑页面，删除之前先确认，操作列固定显示在右侧（fixed="right"）
- 列表表格数据的下方是分页信息，首页、上一页、下一页、尾页，可输入指定页码点击GO跳转页面，可以根据下拉列表随时切换分页大小（这个从视图中配置的分页大小选项配置获取）

## 列表设置页面连接

- 基本信息
    - 编辑列表的名称（必填，不能超过16个字符）、标识（只读）、内容类型（只读）、描述（不能超过255个字符）
- 字段集合，点击后跳转到列表的字段集合页面，与内容类型的字段集合页面一致，列表可以添加（扩展）自己的字段（不影响内容类型），扩展的字段不能跟现有的字段名称和key重复
- 视图集合，点击后跳转到列表的视图集合页面，可以在此新增、编辑、删除视图，默认视图不可删除
    - 新增视图表单，可以指定视图的名称，描述，要显示哪些字段（把所有的字段从上往下列出来，后面有一个复选框，勾选的说明要显示该字段）、可以对要显示的字段进行排序、以及复杂的查询条件（嵌套的AND/OR组合，可以新增条件组，组内需要选择字段，逻辑操作符[等于，大于，小于，大于等于，小于等于，包含]，值 [具体的数字、字符串、今天（等同于当前日期] 2组条件需要选择 AND/OR）、排序（默认按 `created_at` 字段倒序排列）、分页大小选项配置，使用多行文本，使用换行区分，只能输入数字，可以标记是否为默认视图，默认视图只能有1个
    - 编辑同新增
- 表单设置
    - 默认没有组
    - 可以新建分组，然后往组内添加字段，组内可以设置布局，例如每行显示`几列`，使用下拉列表实现，选项有2、3、4、6，哪一列渲染哪些字段
        例如：
        - 新建一个分组，填写名称，指定每行显示3列，然后可以添加列表内的字段，按顺序放入到相应的列中，列中如果有字段则右上角会有一个删除按钮，点击后从组内移除该字段
        - 每一个列可以自己决定需要占多少列，默认值为1，类似合并单元格，但是数字不能超过组设置`几列`的数字, 如果组内设置显示3列，那么列合并单元格的最大值就是3
        - 如果配置好了列，又重新更改了显示`几列`, 则把所有的列的占多少列的值重置为1

## Schema 数据格式

`List.schema` 完整结构如下：

```json
{
    "fields": [
        {
            "id": "uuid-ext-1",
            "name": "自定义字段",
            "key": "custom_field",
            "field_type": "text",
            "required": false,
            "unique": false,
            "searchable": true,
            "order": 0,
            "config": {},
            "validators": []
        }
    ],
    "views": [
        {
            "id": "6eb18686-dfe1-42f1-b1df-0bf6b2a76876",
            "name": "默认视图",
            "is_default": true,
            "page_size_options": [10, 20, 50, 100],
            "default_page_size": 20,
            "fields": [
                { "key": "name", "visible": true },
                { "key": "age", "visible": false }
            ],
            "where": {
                "logic": "OR",
                "left": {
                    "field": "name",
                    "comparison": "=",
                    "value": "1"
                },
                "right": {
                    "field": "name",
                    "comparison": "<>",
                    "value": "2"
                }
            },
            "orderBy": [
                { "field": "name", "sort": "DESC" },
                { "field": "age", "sort": "ASC" }
            ]
        }
    ],
    "form": {
        "groups": [
            {
                "name": "基本信息",
                "columns": 3,
                "fields": [
                    { "key": "name", "col_span": 2 },
                    { "key": "age", "col_span": 1 }
                ]
            }
        ]
    }
}
```

### fields（扩展字段）

- 仅包含列表自行扩展的字段，继承自内容类型的字段不在此数组中
- 每条记录包含 `id`（UUID），PUT 保存时后端根据 ID diff

### views（视图配置）

- `is_default`：是否默认视图，每个列表有且仅有一个默认视图
- `page_size_options`：分页大小候选值
- `default_page_size`：默认分页大小
- `fields`：要显示的字段列表及其可见性，数组顺序即表格列顺序
- `where`：筛选条件，两种节点类型（后端通过是否包含 `logic` 字段判别）：
  - **分支节点**：包含 `logic`（AND/OR）、`left`、`right`。顶层 `OR` 自动用 `UNION ALL` 优化
  - **叶子节点**：包含 `field`、`comparison`、`value`。`comparison` 支持：`=` `<>` `!=` `>` `>=` `<` `<=` `包含`
- `orderBy`：排序规则，数组顺序即排序优先级

### form（表单布局）

- `groups`：表单分组数组，按顺序渲染
- `columns`：每行显示几列
- `fields`：组内字段，`col_span` 为该字段占的列数；`col_span` 不可超过 `columns`

## 动态数据表

每个列表对应一张动态表 `dyn_{list.key}`，表结构：

```sql
CREATE TABLE "dyn_{key}" (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    data JSONB NOT NULL,
    is_deleted BOOLEAN NOT NULL DEFAULT FALSE,
    created_at TIMESTAMPTZ DEFAULT NOW(),
    updated_at TIMESTAMPTZ DEFAULT NOW()
);
CREATE INDEX "idx_{table}_is_deleted" ON "{table}" (is_deleted) WHERE is_deleted = FALSE;
```

- 所有字段值存储在 `data` JSONB 列中
- `is_deleted` 为真实列（非 JSONB 内），配合部分索引加速非删除过滤
- 新增字段时自动创建索引：
  - 文本类（text/long_text）：`GIN ((data->>'key') gin_trgm_ops)`
  - 其他类型：`btree ((data->>'key'))`
- 删除字段时自动删除对应索引
- 新建列表时为内容类型的继承字段自动建索引

## 查询优化

- `WHERE` 顶层 `OR` 条件自动拆为 `UNION ALL`，使每个分支走独立索引
- `ORDER BY` 系统字段（`id`、`created_at`、`updated_at`）使用表列排序，用户字段使用 `data->>'key'` 排序
- 软删除记录通过 `is_deleted = FALSE` 过滤，利用部分索引

## 列表表单

- 根据列表表单设置的分组的配置，按顺序渲染表单中的字段，可以用`el-row`, `el-col :span="24 / 几列 * 占多少列"`实现
- 字段表单渲染参考 `内容类型.md`中 `字段在表单中的外观`, 可以按照这个说明的数量创建新的组件实现
- 字段的验证可以按照列表字段集合中声明的唯一、必填、最大值、最小值等属性来验证，参考`内容类型.md`中各字段类型定义的验证规则

## 列表详情页面

- 布局跟表单页面一样
- 字段都是只读状态
- 页面上有返回按钮，点击后返回列表页面

## API 端点

> 基础路径前缀 `/api`

### 列表 CRUD

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/apps/{appId}/lists/` | 获取应用下所有列表 |
| POST | `/apps/{appId}/lists/` | 创建列表 |
| GET | `/apps/{appId}/lists/{listId}/` | 获取列表详情（含 `schema`） |
| PUT | `/apps/{appId}/lists/{listId}/` | 更新列表基本信息 |
| DELETE | `/apps/{appId}/lists/{listId}/` | 软删除列表 |

### 列表 Schema（字段 + 视图 + 表单布局）

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/apps/{appId}/lists/{listId}/schema/` | 获取完整 schema：`{ fields: [...], views: [...], form: {...} }` |
| PUT | `/apps/{appId}/lists/{listId}/schema/` | 整体保存 schema，后端自动 diff 处理增删改排序 |

> `fields` 为扩展字段（不含继承字段），继承字段在 GET 时合并返回但 PUT 时不提交。
> 前端在本地维护字段/视图/表单布局的状态，点击保存时整体 PUT。

### 表单验证规则

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/apps/{appId}/lists/{listId}/form_schema/` | 读取 schema 后经 ValidationEngine 生成表单字段定义 + Element Plus 验证规则 |

### 记录 CRUD

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/apps/{appId}/lists/{listId}/records/` | 分页查询记录列表 |
| POST | `/apps/{appId}/lists/{listId}/records/` | 新增记录 |
| GET | `/apps/{appId}/lists/{listId}/records/{recordId}/` | 获取单条记录（查看页用） |
| PUT | `/apps/{appId}/lists/{listId}/records/{recordId}/` | 更新记录 |
| DELETE | `/apps/{appId}/lists/{listId}/records/{recordId}/` | 软删除单条记录 |
| PATCH | `/apps/{appId}/lists/{listId}/records/batch/` | 批量操作 `{ action: "delete", ids: [...] }` |

### 记录列表查询参数

| 参数 | 类型 | 说明 |
|------|------|------|
| `view` | string | 视图 `url_key`，指定后使用视图预配置的筛选和排序 |
| `page` | int | 页码，默认 1 |
| `page_size` | int | 每页数量，默认从视图配置获取 |
| `search` | string | 全局搜索关键词 |
| `where` | JSON string | 动态筛选条件，与视图的 `where` 合并（AND） |
| `order_by` | string | 临时排序 `field_key:asc,field_key2:desc`，优先级高于视图排序 |

### 回收站

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/apps/{appId}/trash/` | 获取回收站列表（列表 + 记录） |
| POST | `/apps/{appId}/trash/{itemId}/restore/` | 恢复 |
| DELETE | `/apps/{appId}/trash/{itemId}/` | 永久删除 |
